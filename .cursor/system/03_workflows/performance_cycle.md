# Performance Cycle Workflow

**Modules Involved**: Perform, People

---

## Performance Review Cycle Flow

### Phase 1: Admin Creates a Cycle

1. **Navigate**: Perform → Cycles → "Create Cycle"
2. **Fill cycle details**:
   - Cycle name (e.g., "Q4 2025 Review")
   - Review period: start and end dates
   - Review type: 360°, Top-Down, Self-Review, Peer Review
   - Who is being reviewed: All employees, specific department, specific team
3. **Configure review form**:
   - Add competency sections
   - Add goal review sections
   - Add open-ended questions
   - Set rating scale (1-5, 1-10, etc.)
4. **Set timeline**:
   - When self-reviews open/close
   - When peer reviews open/close
   - When manager reviews open/close
5. **Launch cycle** → Status changes to "Active"

### Verification Points
- ✅ Cycle appears in active cycles list
- ✅ Employees included in the cycle receive email notification
- ✅ Review form appears correctly for each reviewer type

---

### Phase 2: Self-Review (Employee)

1. **Employee receives** notification email
2. **Employee navigates**: Perform → "My Reviews" → Opens self-review
3. **Employee completes** each section (competencies, goals, questions)
4. **Employee submits** → Status: "Self-review Submitted"
5. **Manager is notified** that self-review is complete

### Verification Points
- ✅ Self-review form shows correct sections
- ✅ Employee can save and come back before submitting
- ✅ After submission, employee cannot edit (unless admin unlocks)
- ✅ Manager can see employee's self-review after submission

---

### Phase 3: Peer Review (if included)

1. **Employee selects peers** (if self-selection is enabled) OR admin assigns peers
2. **Peers receive** email invitation to review
3. **Peers complete** review form for the employee
4. **Peer reviews aggregate** in admin/manager view

### Verification Points
- ✅ Peer see only the review questions assigned to them
- ✅ Peer reviewer's identity is anonymous (if anonymous setting is enabled)
- ✅ Peer review results aggregated and shown in manager's view

---

### Phase 4: Manager Review

1. **Manager navigates**: Perform → Reviews (or notification email direct link)
2. **Manager views** employee's self-review (and peer reviews if applicable)
3. **Manager completes** review sections:
   - Rate competencies + provide comments
   - Rate goal achievement
   - Overall rating
   - Manager overall comments
4. **Manager submits** → Review ready to share

### Verification Points
- ✅ Manager can see self-review before writing their review
- ✅ Manager overall rating updates the employee's performance score
- ✅ Review cannot be edited after submission (unless admin unlocks)

---

### Phase 5: Share Results with Employee

1. **Admin/Manager shares** results:
   - Automatic (if auto-share is configured)  
   - Manual: Perform → Cycle → Select employee → "Share Review"
2. **Employee receives** notification
3. **Employee views** their review results in Perform → My Reviews
4. **Employee countersigns** (if signature is required)

### Verification Points
- ✅ Employee can only see review after admin shares it
- ✅ Countersignature prompt appears after sharing
- ✅ Both signatures appear on downloaded review PDF

---

## Goal Tracking Flow

### Setting Goals
1. Navigate: Perform → Goals → "Create Goal"
2. Fill: title, description, due date, measurement type
3. Align to company goal (optional OKR alignment)
4. Assign to employee

### Updating Goal Progress
1. Employee or manager opens goal
2. Updates progress: % complete or status (On Track / At Risk / Behind / Completed)
3. Adds progress notes
4. Goal progress visible in performance review

### Verification Points
- ✅ Goal visible in performance review form
- ✅ Progress updates reflected in review
- ✅ Company-aligned goals show in OKR view

---

## 9-Box Matrix (Talent Review)

1. **Admin configures** X and Y axis criteria (e.g., Performance vs. Potential)
2. After cycle closes, admin views matrix in Perform → Analytics → 9-Box
3. Employees plotted based on their review scores
4. Admin can manually adjust an employee's box placement
5. Export matrix to PDF/Excel

### Verification Points
- ✅ Employees placed in correct box based on their ratings
- ✅ Manual override of placement works
- ✅ Export includes all employees in the cycle
