# Onboard Portals, I-9, and Smartflow

Use this topic for portal-specific behavior, I-9 handling, and advanced automation in Onboard.

## I-9 Specifics

- I-9 is the US employment eligibility verification form.
- After the employee fills Section 1, the employer must complete Section 2.
- If I-9 was submitted with errors, an admin can unlock it for re-editing.
- Employees can also request an unlock.
- E-Verify can be initiated from I-9 to verify work authorization.

## Prehire Portal

- Employees see a portal UI before their start date.
- Portal branding can be customized with logo, colors, and welcome message.
- Multiple portals can be created for different departments or locations.
- Portal display order can be configured.

## Smartflow

- Smartflow is the advanced automation builder inside Onboard.
- It combines conditions, triggers, and actions.
- Example: if `Department = Engineering`, automatically add IT setup tasks.

## Buddy Program

- A buddy can be assigned to support a new hire.
- The buddy receives their own onboarding-related tasks.

## Watchers

- Watchers are additional notified users.
- They are not the assignee; they only receive status notifications.

## Bulk Onboarding

- Multiple prehires can be onboarded at once via bulk action.
